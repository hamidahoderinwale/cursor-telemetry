PKL Dashboard - How It Works

Overview

The PKL Dashboard provides advanced analytics and visualization for Cursor IDE sessions using hierarchical clustering and UMAP dimensionality reduction.

Key Features

  • Hierarchical Clustering: Groups sessions by intent and workflow patterns
  • UMAP Visualization: Shows session relationships in 2D space
  • Pattern Insights: Identifies success patterns and efficiency metrics
  • Real-time Updates: Live monitoring of active sessions
  • Session Details: Deep dive into individual session data

How It Works

  1. Data Collection: Monitors Cursor IDE sessions, conversations, and file changes
  2. Intent Classification: Uses AI to classify session intents (exploration, implementation, debugging)
  3. Clustering: Groups similar sessions using hierarchical clustering algorithms
  4. UMAP Projection: Reduces high-dimensional session data to 2D for visualization
  5. Pattern Analysis: Identifies successful workflow patterns and efficiency metrics

Technical Specifications

Clustering Algorithm: Hierarchical Agglomerative Clustering
Dimensionality Reduction: UMAP (Uniform Manifold Approximation and Projection)
Intent Classification: Custom AI model trained on session patterns
Real-time Processing: Event-driven architecture with WebSocket updates
Data Storage: SQLite database with JSON metadata
Visualization: Plotly.js for interactive charts and D3.js for tree structures

Usage Tips

  • Click on cluster nodes to expand/collapse hierarchical views
  • Use the UMAP plot to explore session relationships
  • Select points or draw selection boxes to see detailed information
  • Change color coding to explore different session attributes
  • Use pattern insights to identify successful workflow strategies
